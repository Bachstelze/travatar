travatar TODO

Travatar is a new decoder, so there are still a number of things that are yet to be implemented. Here is a short list, in approximate order of priority:

  • Implementation of MERT with corpus-level BLEU using all travatar code. (Assigned to Graham)
  • Forest pruning code so that forests with language model integration can still be output in reasonable space, which will allow for forest-based MERT. (Assigned to Graham)
  • Support for parallelization in the decoder.
  • Finish the implementation of online learning algorithms such as MIRA.
  • Implementation of a translation scorer (BLEU, RIBES).
  • On-disk storage of the rule table and language model.
  • More efficient storage of the rule table in memory.
  • Support for minimum Bayes risk (MBR) decoding.
  • Better rule table filtering code.
  • Speed improvements for the tree scoring code.

If you are interested in helping to implement any of these, we'd love to have your help! Please send mail to the travatar-users mailing list, which you can join by sending mail to travatar-users+subscribe with the domain googlegroups.com. (You can also send questions to neubig at gmail.com.)
